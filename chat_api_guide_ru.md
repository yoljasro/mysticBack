# Руководство по Chat API (Mystic)

В этом документе описаны все эндпоинты и события Socket.io для работы чата.

## 🔑 Авторизация
Все запросы требуют JWT токен в заголовке:
`Authorization: Bearer <TOKEN>`

---

## 📂 Эндпоинты (REST API)

### 1. Поиск пользователей
Используется для поиска людей, чтобы начать новый чат.
- **URL:** `/api/auth/users?search=имя_или_телефон`
- **Метод:** `GET`
- **Ответ:** Список найденных пользователей.

### 2. Доступ к чату (1-на-1)
Создает новый чат или открывает существующий с конкретным пользователем.
- **URL:** `/api/chat`
- **Метод:** `POST`
- **Тело (JSON):** `{ "userId": "ID_собеседника" }`

### 3. Получение списка всех чатов
Возвращает все чаты текущего пользователя, отсортированные по последнему сообщению.
- **URL:** `/api/chat`
- **Метод:** `GET`

### 4. Групповые чаты
- **Создать группу:** `POST /api/chat/group` (Тело: `{ "name": "Имя", "users": "[\"ID1\", \"ID2\"]" }`)
- **Переименовать:** `PUT /api/chat/rename` (`{ "chatId": "...", "chatName": "New Name" }`)
- **Добавить участника:** `PUT /api/chat/groupadd` (`{ "chatId": "...", "userId": "..." }`)
- **Удалить участника:** `PUT /api/chat/groupremove` (`{ "chatId": "...", "userId": "..." }`)

### 5. Сообщения
- **Получить историю:** `GET /api/chat/messages/:chatId`
- **Отправить сообщение:** 
    - **URL:** `/api/chat/messages`
    - **Метод:** `POST`
    - **Тип:** `multipart/form-data` (если есть файлы) или `application/json`
    - **Поля:** `chatId`, `content`, `attachments` (файлы).

### 6. Удаление и Очистка
- **Удалить весь чат:** `DELETE /api/chat/:chatId`
- **Очистить сообщения:** `DELETE /api/chat/messages/clear/:chatId` (чат остается, история удаляется).

## ⚡ События Real-time (Socket.io)

### Как подключиться:
```javascript
const socket = io("http://your-server-url");
```

### 1. Вход (Setup)
Должен быть вызван сразу после подключения.
- **Event:** `setup`
- **Data:** `{ "_id": "ID_ТЕКУЩЕГО_ПОЛЬЗОВАТЕЛЯ" }`

### 2. Вход в чат (Join Chat)
Вызывайте, когда пользователь открывает конкретный чат.
- **Event:** `join chat`
- **Data:** `"ID_ЧАТА"`

### 3. Отправка сообщения
После успешного `POST /api/chat/messages` вызовите это событие, чтобы другие увидели сообщение мгновенно.
- **Event:** `new message`
- **Data:** Объект сообщения (который вернул сервер).

### 4. Получение сообщения (Слушать)
- **Event:** `message received`
- **Callback:** `(newMessage) => { ... }`

### 5. Индикаторы печати (Typing)
- **Начать печатать:** `socket.emit("typing", chatId)`
- **Перестать печатать:** `socket.emit("stop typing", chatId)
- **Слушать других:** события `typing` и `stop typing`.
